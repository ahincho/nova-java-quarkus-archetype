# ${artifactId}

Nova Platform Quarkus microservice instance, generated by the
[`ahincho/nova-java-quarkus-archetype`](https://github.com/ahincho/nova-java-quarkus-archetype)
Maven archetype.

- **Stack**: Java 25 + Quarkus 3.33.2.1 LTS
- **Parent**: `pe.edu.nova.java:nova-quarkus-parent:${nova-parent-version}`
- **Notifications**: `pe.edu.nova.java.starters:nova-notifications-quarkus-extension:1.1.2` (pre-included)

## Modules

| Module | Depends on | Purpose |
|---|---|---|
| `${artifactId}-shared` | (none) | Pure-Java domain kernel: `AggregateId`, `DomainException` + `ErrorCode`, `NotFoundException`, `ValidationException`, `ConflictException` |
| `${artifactId}-product` | shared | Example bounded context: `Product` aggregate + `ProductRepository` port + `CreateProductUseCase` |
| `${artifactId}-boot` | shared, product | Quarkus entry-point: `Application`, `ProductResource` REST, `DomainExceptionMapper`, in-memory `ProductRepository` adapter |

## Build

```bash
./mvnw clean verify
```

The `verify` phase runs: `compile + test (Surefire) + integration test (Failsafe, *IT.java) + package + quarkus:build`. The output `boot/target/quarkus-app/quarkus-run.jar` is the runnable artifact.

## Run

```bash
./mvnw -pl boot quarkus:dev
```

Then:

```bash
# List products
curl http://localhost:8080/api/products
# -> []

# Create a product
curl -X POST http://localhost:8080/api/products \
     -H 'Content-Type: application/json' \
     -d '{"name":"Nova Notebook","price":"12.50"}'
# -> 201 {"id":"...","name":"Nova Notebook","price":"12.50"}

# Find by id
curl http://localhost:8080/api/products/<id>
# -> {"id":"...","name":"Nova Notebook","price":"12.50"}

# OpenAPI UI
open http://localhost:8080/q/swagger-ui
# Health
curl http://localhost:8080/q/health
```

## Configuration

`boot/src/main/resources/application.properties`:

```properties
quarkus.http.port=8080
quarkus.http.test-port=8081

# Nova Platform notifications — wired by the extension
nova.notifications.enabled=true
nova.notifications.email.provider=sendgrid
nova.notifications.email.api-key=${SENDGRID_API_KEY:test-api-key-local}
nova.notifications.email.default-sender=no-reply@example.com
nova.notifications.resilience.max-attempts=1
```

Override at runtime with environment variables: `SENDGRID_API_KEY`, `QUARKUS_HTTP_PORT`, etc.

## Docker

```bash
# JVM image (~150 MB)
docker buildx build --platform linux/amd64 -f boot/Dockerfile.jvm \
    -t ${artifactId}:1.0.0 .

# Native image (~50 MB, requires GraalVM setup on the build host)
docker buildx build --platform linux/amd64 -f boot/Dockerfile.native \
    -t ${artifactId}:1.0.0 .

docker run --rm -p 8080:8080 -e SENDGRID_API_KEY=<key> ${artifactId}:1.0.0
```

Both Dockerfiles use:
- `eclipse-temurin:25-{jdk,jre}-alpine` (JVM) or `quarkus-micro-image:2.0` (native)
- non-root UID 1001
- `tini` for proper signal handling
- `-XX:MaxRAMPercentage=75.0` for container-aware heap sizing
- OCI labels

## Tests

```bash
./mvnw test                   # unit tests (Surefire)
./mvnw verify                 # unit + integration tests (Failsafe, *IT.java)
```

The archetype ships with a Quarkus integration test (`ApplicationTest`) that boots the app on the test port and exercises every endpoint:

- `GET /api/products` -> 200 []
- `POST /api/products` -> 201 with body
- `GET /api/products/{id}` -> 200
- `POST /api/products {name:""}` -> 400 VALIDATION
- `GET /api/products/00000000-...` -> 404 NOT_FOUND

## How to add a new bounded context

1. Create a new Maven module under `${artifactId}/` with the same structure as `product/`.
2. Add `<module>your-context</module>` to the root `pom.xml`.
3. Depend on `${artifactId}-shared` for the domain primitives.
4. In `boot/`, add a new `@ApplicationScoped` adapter class that implements your context's port (mirroring `InMemoryProductRepository`).
5. Add a new `@Path("/api/your-context")` resource class. CDI will pick up the adapter automatically.

## How to regenerate this project

If you ever need to re-run the archetype on this directory:

```bash
# From an empty parent dir:
mvn -B archetype:generate \
    -DarchetypeGroupId=pe.edu.nova.java \
    -DarchetypeArtifactId=nova-quarkus-archetype \
    -DarchetypeVersion=1.0.0 \
    -DgroupId=com.example \
    -DartifactId=${artifactId} \
    -Dversion=1.0.0-SNAPSHOT \
    -Dpackage=com.example \
    -Dnova-parent-version=1.0.0
```
